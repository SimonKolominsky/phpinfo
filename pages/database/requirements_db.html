  • Правильное индексирование:

    • Использование логов медленных запросов.

2. MySQL-специфичные знания

Типы хранилищ (engines)

  • Сравнение и применение:

    • InnoDB (транзакции, внешние ключи).

    • MyISAM (полнотекстовый поиск, отсутствие транзакций).

    • Memory (хранение в оперативной памяти).

Оптимизация

  • Настройка конфигурации (my.cnf):

    • Размер буферов (innodb_buffer_pool_size).

    • Настройка кэша запросов (query_cache).

  • Оптимизация соединений и потоков.

Инструменты

  • MySQL Workbench.

  • Percona Toolkit.

  • ProxySQL для балансировки нагрузки.

Типы данных

  • Оптимальное использование типов:

    • CHAR vs VARCHAR.

    • INT vs BIGINT.

  • Поддержка JSON (начиная с MySQL 5.7).

Расширенные возможности

  • Полнотекстовый поиск:

    • tsvector, tsquery.

  • Геопространственные данные:

    • Расширение PostGIS.

  • Нативная поддержка JSON (JSONB).

  • Наследование таблиц.

Типы данных

  • Пользовательские типы данных.

  • Массивы и составные типы.

  • Range типы данных.

  • Enum типы.

Особенности работы

  • VACUUM и автовакуум.

  • Последовательности (SEQUENCE).

  • Правила (RULES) и триггеры.

  • Оконные функции (Window Functions).

Расширения

  • pg_stat_statements для мониторинга.

  • pgBadger для анализа логов.

  • Партиционирование с использованием pg_partman.
