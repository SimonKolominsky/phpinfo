SQL (Structured Query Language) — это язык программирования, используемый для работы с реляционными базами данных. Он позволяет создавать, изменять, управлять и извлекать данные из баз данных. Вот базовая информация о SQL:

Основные понятия

  1. База данных (Database) — организованная коллекция данных, хранящихся в структурированном виде.

  2. Таблица (Table) — структура внутри базы данных, которая хранит данные в виде строк (записей) и столбцов (полей).

  3. Строка (Row) — запись в таблице, представляющая собой набор данных.

  4. Столбец (Column) — поле в таблице, которое хранит определённый тип данных (например, имя, возраст, дата).

  5. Первичный ключ (Primary Key) — уникальный идентификатор для каждой строки в таблице.

  6. Внешний ключ (Foreign Key) — поле, которое ссылается на первичный ключ другой таблицы для обеспечения связей между таблицами.

Основные команды SQL

  1. SELECT — извлечение данных из таблицы.

    sql
    Copy
    SELECT column1, column2 FROM table_name;

    Пример:

    sql
    Copy
    SELECT name, age FROM users;

  2. INSERT INTO — добавление новых строк в таблицу.

    sql
    Copy
    INSERT INTO table_name (column1, column2) VALUES (value1, value2);

    Пример:

    sql
    Copy
    INSERT INTO users (name, age) VALUES ('Alice', 25);

  3. UPDATE — обновление существующих данных в таблице.

    sql
    Copy
    UPDATE table_name SET column1 = value1 WHERE condition;

    Пример:

    sql
    Copy
    UPDATE users SET age = 26 WHERE name = 'Alice';

  4. DELETE — удаление строк из таблицы.

    sql
    Copy
    DELETE FROM table_name WHERE condition;

    Пример:

    sql
    Copy
    DELETE FROM users WHERE name = 'Alice';

  5. CREATE TABLE — создание новой таблицы.

    sql
    Copy
    CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    ...
);

    Пример:

    sql
    Copy
    CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);

  6. ALTER TABLE — изменение структуры существующей таблицы (добавление, удаление или изменение столбцов).

    sql
    Copy
    ALTER TABLE table_name ADD column_name datatype;

    Пример:

    sql
    Copy
    ALTER TABLE users ADD email VARCHAR(100);

  7. DROP TABLE — удаление таблицы.

    sql
    Copy
    DROP TABLE table_name;

    Пример:

    sql
    Copy
    DROP TABLE users;

  8. JOIN — объединение данных из двух или более таблиц на основе связанных столбцов.

    • INNER JOIN: Возвращает строки, которые имеют совпадения в обеих таблицах.

    • LEFT JOIN: Возвращает все строки из левой таблицы и совпадающие строки из правой таблицы.

    • RIGHT JOIN: Возвращает все строки из правой таблицы и совпадающие строки из левой таблицы.

    • FULL JOIN: Возвращает строки, если есть совпадение в одной из таблиц.

    Пример:

    sql
    Copy
    SELECT users.name, orders.order_id
FROM users
INNER JOIN orders ON users.id = orders.user_id;

Фильтрация и сортировка

  1. WHERE — фильтрация строк по условию.

    sql
    Copy
    SELECT * FROM users WHERE age > 20;

  2. ORDER BY — сортировка результатов по одному или нескольким столбцам.

    sql
    Copy
    SELECT * FROM users ORDER BY age DESC;

  3. LIMIT — ограничение количества возвращаемых строк.

    sql
    Copy
    SELECT * FROM users LIMIT 10;

Агрегатные функции

  1. COUNT() — подсчёт количества строк.

    sql
    Copy
    SELECT COUNT(*) FROM users;

  2. SUM() — сумма значений в столбце.

    sql
    Copy
    SELECT SUM(age) FROM users;

  3. AVG() — среднее значение в столбце.

    sql
    Copy
    SELECT AVG(age) FROM users;

  4. MIN() и MAX() — минимальное и максимальное значение в столбце.

    sql
    Copy
    SELECT MIN(age), MAX(age) FROM users;

Группировка данных

  1. GROUP BY — группировка строк по значениям столбцов.

    sql
    Copy
    SELECT age, COUNT(*) FROM users GROUP BY age;

  2. HAVING — фильтрация результатов группировки.

    sql
    Copy
    SELECT age, COUNT(*) FROM users GROUP BY age HAVING COUNT(*) > 1;

Примеры запросов

  1. Выбор всех пользователей старше 30 лет:

    sql
    Copy
    SELECT * FROM users WHERE age > 30;

  2. Добавление нового пользователя:

    sql
    Copy
    INSERT INTO users (name, age) VALUES ('Bob', 35);

  3. Обновление возраста пользователя:

    sql
    Copy
    UPDATE users SET age = 40 WHERE name = 'Bob';

  4. Удаление пользователя:

    sql
    Copy
    DELETE FROM users WHERE name = 'Bob';

Типы операторов

SQL (Structured Query Language) состоит из нескольких подмножеств, каждое из которых отвечает за определенные аспекты работы с базами данных. Основные подмножества SQL включают:

  1. DDL (Data Definition Language) — язык определения данных:

    • Используется для создания, изменения и удаления структуры базы данных и её объектов (таблиц, индексов, представлений и т.д.).

    • Основные команды:

      • CREATE — создание объектов (например, таблиц).

      • ALTER — изменение структуры объектов.

      • DROP — удаление объектов.

      • TRUNCATE — очистка таблицы (удаление всех данных без удаления самой таблицы).

      • RENAME — переименование объектов.

  2. DML (Data Manipulation Language) — язык управления данными:

    • Используется для работы с данными в таблицах (добавление, изменение, удаление и выборка данных).

    • Основные команды:

      • SELECT — выборка данных.

      • INSERT — добавление новых данных.

      • UPDATE — изменение существующих данных.

      • DELETE — удаление данных.

  3. DCL (Data Control Language) — язык управления доступом:

    • Используется для управления правами доступа к данным и объектам базы данных.

    • Основные команды:

      • GRANT — предоставление прав доступа.

      • REVOKE — отзыв прав доступа.

  4. TCL (Transaction Control Language) — язык управления транзакциями:

    • Используется для управления транзакциями в базе данных.

    • Основные команды:

      • COMMIT — подтверждение транзакции.

      • ROLLBACK — откат транзакции.

      • SAVEPOINT — создание точки сохранения внутри транзакции.

  5. DQL (Data Query Language) — язык запросов данных:

    • Хотя формально DQL не всегда выделяется как отдельное подмножество, его можно рассматривать как часть DML, отвечающую исключительно за выборку данных.

    • Основная команда:

      • SELECT — выборка данных.

Эти подмножества охватывают все основные аспекты работы с базами данных, начиная от создания структуры и заканчивая управлением доступом и транзакциями.
