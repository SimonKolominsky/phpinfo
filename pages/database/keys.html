В MySQL и PostgreSQL используются различные типы ключей для обеспечения целостности данных и организации связей между таблицами. Основные типы ключей в этих СУБД:

1. Первичный ключ (Primary Key)

  • Назначение: Уникально идентифицирует каждую запись в таблице.

  • Особенности:

    • Значение должно быть уникальным и не может быть NULL.

    • В таблице может быть только один первичный ключ.

    • Часто используется для создания индексов.

  • Пример:

    sql
    Copy
    CREATE TABLE users (
    id INT PRIMARY KEY,
    username VARCHAR(50)
);

2. Внешний ключ (Foreign Key)

  • Назначение: Обеспечивает связь между таблицами, ссылаясь на первичный ключ другой таблицы.

  • Особенности:

    • Значение может быть NULL, если это разрешено.

    • Используется для поддержания ссылочной целостности.

  • Пример:

    sql
    Copy
    CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    user_id INT,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

3. Уникальный ключ (Unique Key)

  • Назначение: Гарантирует уникальность значений в столбце или группе столбцов.

  • Особенности:

    • Значение может быть NULL (в зависимости от СУБД).

    • В таблице может быть несколько уникальных ключей.

  • Пример:

    sql
    Copy
    CREATE TABLE products (
    product_id INT PRIMARY KEY,
    sku VARCHAR(50) UNIQUE
);

4. Составной ключ (Composite Key)

  • Назначение: Используется, когда уникальность обеспечивается комбинацией нескольких столбцов.

  • Особенности:

    • Может быть первичным или уникальным ключом.

  • Пример:

    sql
    Copy
    CREATE TABLE orders (
    order_id INT,
    product_id INT,
    PRIMARY KEY (order_id, product_id)
);

5. Суперключ (Super Key)

  • Назначение: Набор столбцов, который уникально идентифицирует запись.

  • Особенности:

    • Может содержать избыточные столбцы.

    • Не обязательно используется в явном виде в SQL.

6. Кандидат на ключ (Candidate Key)

  • Назначение: Минимальный набор столбцов, который может быть использован как первичный ключ.

  • Особенности:

    • В таблице может быть несколько кандидатов на ключ.

    • Один из них выбирается как первичный ключ.

7. Альтернативный ключ (Alternate Key)

  • Назначение: Кандидат на ключ, который не был выбран в качестве первичного ключа.

  • Особенности:

    • Часто используется как уникальный ключ.

8. Естественный ключ (Natural Key)

  • Назначение: Использует существующие данные (например, email, паспортные данные) для уникальной идентификации.

  • Особенности:

    • Может изменяться со временем, что усложняет поддержку.

9. Суррогатный ключ (Surrogate Key)

  • Назначение: Искусственно созданный ключ (например, автоинкрементный id).

  • Особенности:

    • Не имеет смысловой нагрузки.

    • Упрощает управление данными.

Особенности в MySQL и PostgreSQL

  • MySQL:

    • Поддерживает все основные типы ключей.

    • Автоинкремент (AUTO_INCREMENT) часто используется для суррогатных ключей.

  • PostgreSQL:

    • Также поддерживает все основные типы ключей.

    • Для автоинкремента используется тип SERIAL или IDENTITY.
