PSR (PHP Standards Recommendations) — это набор стандартов и рекомендаций для языка программирования PHP, разработанных группой PHP-FIG (Framework Interop Group). Эти стандарты созданы для улучшения взаимодействия между различными PHP-проектами и фреймворками, а также для повышения качества кода.

Основные PSR:

  1. PSR-1: Basic Coding Standard:

    • Основные стандарты написания кода на PHP.

    • Примеры:

      • Файлы должны использовать только теги <?php и <?=.

      • Имена классов должны быть в стиле StudlyCaps.

      • Константы должны быть в верхнем регистре с подчёркиваниями.

  2. PSR-2: Coding Style Guide:

    • Расширение PSR-1, которое определяет стиль написания кода.

    • Примеры:

      • Использование 4 пробелов для отступов.

      • Открывающая фигурная скобка { должна быть на той же строке, что и объявление класса или метода.

      • Ключевые слова должны быть в нижнем регистре.

  3. PSR-3: Logger Interface:

    • Стандартизирует интерфейс для логгеров.

    • Пример использования:

      php
      Copy
      $logger->info('User logged in', ['username' => 'john_doe']);

  4. PSR-4: Autoloading Standard:

    • Стандарт для автозагрузки классов на основе пространств имён (namespaces).

    • Пример:

      • Пространство имён App\Core должно соответствовать директории src/Core.

      • Файл App\Core\Controller должен находиться в src/Core/Controller.php.

  5. PSR-6: Caching Interface:

    • Стандартизирует интерфейс для работы с кэшированием.

    • Пример:

      php
      Copy
      $cacheItem = $cache->getItem('cache_key');
$cacheItem->set('value');
$cache->save($cacheItem);

  6. PSR-7: HTTP Message Interface:

    • Определяет интерфейсы для работы с HTTP-сообщениями (запросы и ответы).

    • Пример:

      php
      Copy
      $response = new Response();
$response->getBody()->write('Hello, World!');

  7. PSR-11: Container Interface:

    • Стандартизирует интерфейс для контейнеров внедрения зависимостей (Dependency Injection).

    • Пример:

      php
      Copy
      $container->get('service_name');

  8. PSR-12: Extended Coding Style Guide:

    • Обновлённая версия PSR-2 с дополнительными правилами.

    • Пример:

      • Использование strict-типизации (declare(strict_types=1)).

      • Пустые строки вокруг тел методов и функций.

  9. PSR-15: HTTP Server Request Handlers:

    • Стандартизирует обработку HTTP-запросов с использованием middleware.

    • Пример:

      php
      Copy
      $handler = new RequestHandler();
$response = $handler->handle($request);

  10. PSR-18: HTTP Client:

    • Стандартизирует интерфейс для HTTP-клиентов.

    • Пример:

      php
      Copy
      $response = $client->sendRequest($request);

Зачем нужны PSR?

  1. Совместимость:

    • PSR позволяют разным библиотекам и фреймворкам работать вместе.

  2. Единый стиль кода:

    • Упрощает чтение и поддержку кода.

  3. Стандартизация:

    • Уменьшает количество споров о том, как писать код.

Как использовать PSR?

  1. Установите инструменты для проверки стиля кода, такие как:

    • PHP_CodeSniffer для проверки PSR-1, PSR-2, PSR-12.

    • Composer для автозагрузки классов по PSR-4.

  2. Следуйте рекомендациям PSR при написании кода.
