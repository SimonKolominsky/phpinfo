SOLID — это набор принципов объектно-ориентированного программирования, которые делают код более гибким, понятным и поддерживаемым. Рассмотрим каждый принцип с примерами на PHP.

1. Принцип единственной ответственности (Single Responsibility Principle, SRP)

Суть: Класс должен иметь только одну причину для изменения (одну ответственность).

Пример нарушения:

php
Copy
class Order {
    public function calculateTotalSum() { /* ... */ }
    public function saveToDatabase() { /* ... */ }
    public function sendEmail() { /* ... */ }
}

Здесь класс Order отвечает за расчет суммы, сохранение в БД и отправку email — это три разные ответственности.

Исправленный код:

php
Copy
class Order {
    public function calculateTotalSum() { /* ... */ }
}

class OrderRepository {
    public function save(Order $order) { /* ... */ }
}

class EmailNotifier {
    public function sendEmail(Order $order) { /* ... */ }
}

Теперь каждый класс решает свою задачу.

2. Принцип открытости/закрытости (Open-Closed Principle, OCP)

Суть: Классы должны быть открыты для расширения, но закрыты для модификации.

Пример нарушения:

php
Copy
class AreaCalculator {
    public function calculate($shapes) {
        $area = 0;
        foreach ($shapes as $shape) {
            if ($shape instanceof Circle) {
                $area += $shape->radius * $shape->radius * pi();
            } elseif ($shape instanceof Square) {
                $area += $shape->side * $shape->side;
            }
            // Добавление новой фигуры требует изменения класса!
        }
        return $area;
    }
}

Исправленный код:

php
Copy
interface Shape {
    public function area();
}

class Circle implements Shape {
    public $radius;
    public function area() {
        return $this->radius * $this->radius * pi();
    }
}

class Square implements Shape {
    public $side;
    public function area() {
        return $this->side * $this->side;
    }
}

class AreaCalculator {
    public function calculate(array $shapes) {
        $area = 0;
        foreach ($shapes as $shape) {
            $area += $shape->area();
        }
        return $area;
    }
}

Теперь для добавления новой фигуры (например, Triangle) достаточно реализовать интерфейс Shape, не меняя AreaCalculator.

3. Принцип подстановки Лисков (Liskov Substitution Principle, LSP)

Суть: Объекты производных классов должны заменять объекты базовых классов без нарушения работы программы.

Пример нарушения:

php
Copy
class Rectangle {
    public $width;
    public $height;
    public function setWidth($w) { $this->width = $w; }
    public function setHeight($h) { $this->height = $h; }
}

class Square extends Rectangle {
    public function setWidth($w) {
        $this->width = $w;
        $this->height = $w;
    }
    public function setHeight($h) {
        $this->height = $h;
        $this->width = $h;
    }
}

Квадрат меняет поведение методов setWidth и setHeight, что может сломать код, ожидающий прямоугольник.

Исправленный код:

php
Copy
interface Shape {
    public function getArea();
}

class Rectangle implements Shape {
    public function __construct($width, $height) { /* ... */ }
    public function getArea() { return $this->width * $this->height; }
}

class Square implements Shape {
    public function __construct($side) { /* ... */ }
    public function getArea() { return $this->side * $this->side; }
}

Квадрат не наследуется от прямоугольника, оба реализуют общий интерфейс.

4. Принцип разделения интерфейса (Interface Segregation Principle, ISP)

Суть: Клиенты не должны зависеть от интерфейсов, которые они не используют.

Пример нарушения:

php
Copy
interface Worker {
    public function work();
    public function eat();
}

class HumanWorker implements Worker {
    public function work() { /* ... */ }
    public function eat() { /* ... */ }
}

class RobotWorker implements Worker {
    public function work() { /* ... */ }
    public function eat() { throw new Exception("Робот не ест!"); } // Нарушение ISP
}

Исправленный код:

php
Copy
interface Workable {
    public function work();
}

interface Eatable {
    public function eat();
}

class HumanWorker implements Workable, Eatable {
    public function work() { /* ... */ }
    public function eat() { /* ... */ }
}

class RobotWorker implements Workable {
    public function work() { /* ... */ }
}

Интерфейсы разделены, и классы реализуют только нужные.

5. Принцип инверсии зависимостей (Dependency Inversion Principle, DIP)

Суть: Зависимости должны строиться на абстракциях, а не на деталях.

Пример нарушения:

php
Copy
class Report {
    private $mySQLConnection;
    public function __construct() {
        $this->mySQLConnection = new MySQLConnection(); // Зависимость от конкретного класса
    }
}

Исправленный код:

php
Copy
interface DatabaseConnection {
    public function connect();
}

class MySQLConnection implements DatabaseConnection {
    public function connect() { /* ... */ }
}

class PostgreSQLConnection implements DatabaseConnection {
    public function connect() { /* ... */ }
}

class Report {
    private $dbConnection;
    public function __construct(DatabaseConnection $dbConnection) {
        $this->dbConnection = $dbConnection; // Зависимость от абстракции
    }
}

Теперь Report зависит от интерфейса, а не от конкретной реализации.

Итог

SOLID помогает создавать код, который:

  • Легко расширять и модифицировать.

  • Минимизирует риск внесения ошибок.

  • Упрощает тестирование и повторное использование компонентов.

Примеры на PHP демонстрируют, как применять эти принципы на практике.
