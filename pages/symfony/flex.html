Symfony Flex — это современный инструмент для управления проектами на Symfony, представленный в Symfony 4. Он заменяет старый способ настройки приложений (через стандартные бандлы) и значительно упрощает процесс создания, настройки и поддержки проектов. Flex работает как плагин для Composer, автоматизируя рутинные задачи и предоставляя гибкость в управлении зависимостями.

Основные особенности Symfony Flex:

  1. Рецепты (Recipes)
    Flex использует "рецепты" — наборы конфигураций и файлов, которые автоматически применяются при установке пакетов через Composer. Рецепты хранятся в репозитории symfony/recipes-contrib (для сообщества) и symfony/recipes (официальные рецепты от Symfony).
    Например, при установке пакета annotations Flex автоматически добавит настройки для работы с аннотациями в config/packages/annotations.yaml.

  2. Упрощённая структура проекта
    Symfony Flex убирает избыточную структуру каталогов. Вместо app/, src/, web/ и других папок используется минималистичная структура:

    • config/ — конфигурационные файлы (YAML, XML, PHP).

    • src/ — исходный код приложения.

    • public/ — публичные файлы (входная точка index.php, стили, скрипты).

    • var/ — кэш, логи.

    • vendor/ — зависимости Composer.

  3. Пакеты (Packs)
    Flex вводит концепцию "паков" — мета-пакетов, объединяющих несколько связанных зависимостей. Например:

    • composer require symfony/apache-pack — добавляет настройки для Apache.

    • composer require symfony/orm-pack — устанавливает Doctrine ORM и его интеграцию с Symfony.

  4. Автоматическая настройка
    При установке пакета Flex автоматически:

    • Регистрирует бандлы в config/bundles.php.

    • Создаёт базовые конфигурационные файлы.

    • Добавляет примеры кода (например, контроллеры или шаблоны Twig).

    • Настраивает переменные окружения в .env.

  5. Переменные окружения
    Flex активно использует файл .env для хранения настроек, таких как DATABASE_URL, что упрощает управление конфигурацией для разных сред (разработка, прод).

Пример работы Symfony Flex:

  1. Установка пакета:

    bash
    Copy
    composer require logger

    Flex автоматически:

    • Установит пакет symfony/logger.

    • Добавит бандл MonologBundle в config/bundles.php.

    • Создаст конфигурацию в config/packages/monolog.yaml.

    • Обновит .env для настройки уровня логирования.

  2. Удаление пакета:

    bash
    Copy
    composer remove logger

    Flex удалит связанные конфигурации (если рецепт поддерживает откат).

Преимущества:

  • Сокращение boilerplate-кода — больше не нужно вручную регистрировать бандлы или настраивать пакеты.

  • Стандартизация — рецепты обеспечивают единый подход к настройке зависимостей.

  • Быстрый старт — новые пакеты сразу готовы к использованию.

  • Гибкость — можно переопределить автоматически сгенерированные конфигурации.

Ограничения:

  • Зависимость от рецептов — если рецепта для пакета нет, его придётся настраивать вручную.

  • Несовместимость с Symfony 3 — Flex работает только для Symfony 4 и выше.

Как начать использовать Symfony Flex?

  1. Убедитесь, что у вас установлен Composer 2+.

  2. Создайте проект:

    bash
    Copy
    composer create-project symfony/skeleton:"6.*" my_project

    Готово! Flex активируется автоматически.

Symfony Flex делает разработку на Symfony быстрее и удобнее, особенно для новичков. Он убирает рутину, позволяя сосредоточиться на написании кода, а не на настройке инфраструктуры.
