В PHPUnit, как и в любом фреймворке для тестирования, есть несколько ключевых объектов и концепций, которые используются для создания и выполнения тестов. Вот основные из них:

1. Тестовый класс (TestCase)

  • Описание: Базовый класс для всех тестовых классов. Каждый тестовый класс должен наследоваться от PHPUnit\Framework\TestCase.

  • Пример:

    php
    Copy
    use PHPUnit\Framework\TestCase;

class MyTest extends TestCase {
    public function testSomething() {
        $this->assertTrue(true);
    }
}

2. Тестовые методы

  • Описание: Методы, которые начинаются с префикса test, автоматически распознаются PHPUnit как тестовые методы. Также можно использовать аннотацию @test для указания тестовых методов.

  • Пример:

    php
    Copy
    class MyTest extends TestCase {
    public function testAddition() {
        $this->assertEquals(4, 2 + 2);
    }

    /**
     * @test
     */
    public function addition() {
        $this->assertEquals(4, 2 + 2);
    }
}

3. Утверждения (Assertions)

  • Описание: Утверждения используются для проверки того, что результат выполнения кода соответствует ожидаемому. PHPUnit предоставляет множество методов для утверждений, таких как assertEquals, assertTrue, assertFalse, assertNull, и т.д.

  • Пример:

    php
    Copy
    class MyTest extends TestCase {
    public function testAssertions() {
        $this->assertEquals(4, 2 + 2);
        $this->assertTrue(true);
        $this->assertFalse(false);
        $this->assertNull(null);
    }
}

4. Фикстуры (Fixtures)

  • Описание: Фикстуры используются для настройки среды тестирования перед выполнением тестов и её очистки после выполнения. Методы setUp и tearDown вызываются перед и после каждого тестового метода соответственно.

  • Пример:

    php
    Copy
    class MyTest extends TestCase {
    protected $value;

    protected function setUp(): void {
        $this->value = 2;
    }

    protected function tearDown(): void {
        unset($this->value);
    }

    public function testValue() {
        $this->assertEquals(2, $this->value);
    }
}

5. Провайдеры данных (Data Providers)

  • Описание: Провайдеры данных позволяют передавать различные наборы данных в тестовые методы. Метод-провайдер должен возвращать массив массивов или итератор.

  • Пример:

    php
    Copy
    class MyTest extends TestCase {
    /**
     * @dataProvider additionProvider
     */
    public function testAddition($a, $b, $expected) {
        $this->assertEquals($expected, $a + $b);
    }

    public function additionProvider() {
        return [
            [1, 1, 2],
            [2, 2, 4],
            [3, 3, 6],
        ];
    }
}

6. Моки и стабы (Mocks and Stubs)

  • Описание: Моки и стабы используются для создания объектов-заглушек, которые имитируют поведение реальных объектов. Это полезно для тестирования взаимодействия между объектами.

  • Пример:

    php
    Copy
    class MyTest extends TestCase {
    public function testMock() {
        $mock = $this->createMock(SomeClass::class);
        $mock->method('doSomething')->willReturn('foo');

        $this->assertEquals('foo', $mock->doSomething());
    }
}

7. Группы тестов (Test Groups)

  • Описание: Группы тестов позволяют группировать тестовые методы и выполнять их выборочно. Группы задаются с помощью аннотации @group.

  • Пример:

    php
    Copy
    class MyTest extends TestCase {
    /**
     * @group fast
     */
    public function testFast() {
        $this->assertTrue(true);
    }

    /**
     * @group slow
     */
    public function testSlow() {
        $this->assertTrue(true);
    }
}

8. Исключения (Exceptions)

  • Описание: PHPUnit позволяет тестировать код, который должен выбрасывать исключения. Для этого используется метод expectException.

  • Пример:

    php
    Copy
    class MyTest extends TestCase {
    public function testException() {
        $this->expectException(\InvalidArgumentException::class);
        throw new \InvalidArgumentException();
    }
}

9. Тестовые наборы (Test Suites)

  • Описание: Тестовые наборы позволяют объединять несколько тестовых классов в один набор для выполнения.

  • Пример:

    xml
    Copy
    <phpunit>
    <testsuites>
        <testsuite name="My Test Suite">
            <file>MyTest.php</file>
            <file>AnotherTest.php</file>
        </testsuite>
    </testsuites>
</phpunit>
    Run HTML

10. Аннотации (Annotations)

  • Описание: Аннотации используются для настройки тестов и тестовых методов. Например, @depends, @dataProvider, @group, и т.д.

  • Пример:

    php
    Copy
    class MyTest extends TestCase {
    /**
     * @depends testOne
     */
    public function testTwo() {
        $this->assertTrue(true);
    }

    public function testOne() {
        $this->assertTrue(true);
    }
}

11. Тестовые двойники (Test Doubles)

  • Описание: Тестовые двойники — это объекты, которые заменяют реальные объекты в тестах. К ним относятся моки, стабы, фейки и шпионы.

  • Пример:

    php
    Copy
    class MyTest extends TestCase {
    public function testStub() {
        $stub = $this->createStub(SomeClass::class);
        $stub->method('doSomething')->willReturn('foo');

        $this->assertEquals('foo', $stub->doSomething());
    }
}

Заключение

PHPUnit предоставляет богатый набор инструментов для написания и выполнения тестов. Понимание этих объектов и концепций позволяет эффективно тестировать PHP-код, обеспечивая его качество и надежность.
