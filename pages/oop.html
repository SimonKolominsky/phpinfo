Объектно-ориентированное программирование (ООП)

Объектно-ориентированное программирование (ООП) — это подход в основе которого лежат наследование, инкапсуляция, полиморфизм и в котором программа рассматривается как набор объектов, взаимодействующих друг с другом. У каждого есть свойства и поведение.
Используется, чтобы:

  • Структурировать информацию и не допускать путаницы;

  • Точно определять взаимодействие одних элементов с другими;

  • Повышать управляемость программы;

  • Быстрее масштабировать код под различные задачи;

  • Лучше понимать написанное;

  • Эффективнее поддерживать готовые программы;

  • Внедрять изменения без необходимости переписывать весь код.

Основные принципы ООП

1. Инкапсуляция

Связывание состояния и поведения объекта вместе, защита этого от внешнего вмешательства или неправильного использования.

Пример:

php
Copy
class BankAccount {
    private $balance = 0;

    public function deposit($amount) {
        if ($amount > 0) {
            $this->balance += $amount;
        }
    }

    public function getBalance() {
        return $this->balance;
    }
}

$account = new BankAccount();
$account->deposit(100);
echo $account->getBalance(); // 100

2. Наследование

Создание нового класса на основе существующего с расширением функциональности.

Пример:

php
Copy
class Animal {
    public function makeSound() {
        return "Some sound";
    }
}

class Dog extends Animal {
    public function makeSound() {
        return "Woof!";
    }
}

$dog = new Dog();
echo $dog->makeSound(); // Woof!

3. Полиморфизм

Принцип, по которому объект может изменяться или приобретать новые свойства. Семейство различных механизмов, позволяющих использовать один и тот же участок программы с различными типами в различных контекстах.

Пример через интерфейс:

php
Copy
interface Shape {
    public function area();
}

class Circle implements Shape {
    private $radius;
    public function __construct($radius) { $this->radius = $radius; }
    public function area() { return pi() * $this->radius ** 2; }
}

class Square implements Shape {
    private $side;
    public function __construct($side) { $this->side = $side; }
    public function area() { return $this->side ** 2; }
}

// Полиморфное использование:
$shapes = [new Circle(3), new Square(4)];
foreach ($shapes as $shape) {
    echo $shape->area() . "\n";
}

4. Абстракция

Создание упрощенных моделей сокрытием сложных деталей.

Пример через абстрактный класс:

php
Copy
abstract class Vehicle {
    abstract public function startEngine();
}

class Car extends Vehicle {
    public function startEngine() {
        return "Car engine started!";
    }
}

$car = new Car();
echo $car->startEngine(); // Car engine started!

Дополнительные принципы ООП

Помимо основных принципов, существуют дополнительные концепции, которые часто применяются в ООП:

1. SOLID-принципы

  • S (Single Responsibility): Класс должен иметь одну ответственность.

  • O (Open/Closed): Классы должны быть открыты для расширения, но закрыты для изменения.

  • L (Liskov Substitution): Подклассы должны заменять базовые классы без ошибок.

  • I (Interface Segregation): Много специализированных интерфейсов лучше одного общего.

  • D (Dependency Inversion): Зависимость от абстракций, а не конкретных реализаций.

2. Ассоциация, композиция и агрегация

Ассоциация

Связь между двумя независимыми объектами, которые могут существовать отдельно.
Пример:

php
Copy
class User {
    private $profile;

    // Ассоциация: User связан с Profile, но не управляет его жизненным циклом
    public function setProfile(Profile $profile) {
        $this->profile = $profile;
    }
}

class Profile {
    private $name;

    public function __construct($name) {
        $this->name = $name;
    }
}

// Создаем объекты отдельно
$profile = new Profile("Иван Иванов");
$user = new User();
$user->setProfile($profile); // Устанавливаем связь

Композиция

Объект-часть (зависимый) создается внутри объекта-целого (родителя) и не может существовать без него.
Пример:

php
Copy
class Car {
    private $engine;

    // Композиция: Engine создается внутри Car и зависит от него
    public function __construct() {
        $this->engine = new Engine(); // Зависимый объект создается здесь
    }

    public function start() {
        $this->engine->ignite();
    }
}

class Engine {
    public function ignite() {
        echo "Двигатель запущен!";
    }
}

// Создаем Car, который автоматически создает Engine внутри себя
$car = new Car();
$car->start(); // Выведет: "Двигатель запущен!"

// Если удалить Car, Engine тоже будет уничтожен

Агрегация

Класс получает объект другого независимого класса извне и использует его.
Пример:

php
Copy
class University {
    private $departments = [];

    // Агрегация: Department передается извне и не зависит от University
    public function addDepartment(Department $department) {
        $this->departments[] = $department;
    }
}

class Department {
    private $name;

    public function __construct($name) {
        $this->name = $name;
    }
}

// Создаем Department независимо от University
$mathDepartment = new Department("Математика");
$physicsDepartment = new Department("Физика");

// Передаем Department в University
$university = new University();
$university->addDepartment($mathDepartment);
$university->addDepartment($physicsDepartment);

// Даже если University удалить, Department останутся

Ключевые различия

  1. Композиция:

    • Зависимый объект создается внутри родителя.

    • Жизненный цикл зависимого объекта контролируется родителем.

    • Пример: Car и Engine.

  2. Агрегация:

    • Зависимый объект передается извне.

    • Жизненный цикл зависимого объекта НЕ зависит от родителя.

    • Пример: University и Department.

  3. Ассоциация:

    • Объекты слабо связаны (просто знают друг о друге).

    • Пример: User и Profile.

3. DRY (Don’t Repeat Yourself)

Избегание дублирования кода.

Пример:

php
Copy
// Плохо:
class Logger {
    public function logToFile($message) { /* ... */ }
    public function logToDatabase($message) { /* ... */ }
}

// Хорошо (вынеси общую логику):
abstract class Logger {
    abstract protected function save($message);
    public function log($message) { $this->save($message); }
}

class FileLogger extends Logger { protected function save($message) { /* ... */ } }

4. KISS (Keep It Simple, Stupid)

Код должен быть простым и понятным.

Пример:

php
Copy
// Плохо: избыточная сложность
class OvercomplicatedCalculator {
    public function add($a, $b) {
        // ... 20 строк кода ...
        return $a + $b;
    }
}

// Хорошо:
class Calculator {
    public function add($a, $b) { return $a + $b; }
}

5. Принцип Demeter (Law of Demeter)

Объект не должен взаимодействовать с внутренними структурами других объектов.

Пример:

php
Copy
// Плохо:
$user->getOrder()->getItems()->count();

// Хорошо (делегирование):
class User {
    public function getOrderItemCount() {
        return $this->order->getItemCount();
    }
}
$user->getOrderItemCount();

Итог

Основные принципы ООП в PHP позволяют создавать структурированные и гибкие приложения. Дополнительные принципы (SOLID, DRY, KISS и др.) помогают улучшить читаемость, поддерживаемость и масштабируемость кода. Используйте их в зависимости от контекста задачи.
