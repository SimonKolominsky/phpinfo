Сложность алгоритма — это мера того, сколько времени или ресурсов потребуется алгоритму для выполнения задачи в зависимости от размера входных данных. Она выражается с помощью "нотации Big O" (O-нотация), которая описывает верхнюю границу времени выполнения или использования памяти алгоритма.

Основные виды сложности:

  1. O(1) — Константная сложность. Время выполнения не зависит от размера входных данных.

  2. O(log n) — Логарифмическая сложность. Время выполнения растет логарифмически от размера входных данных.

  3. O(n) — Линейная сложность. Время выполнения растет линейно с увеличением размера входных данных.

  4. O(n log n) — Линейно-логарифмическая сложность. Время выполнения растет пропорционально произведению размера данных на логарифм размера.

  5. O(n²) — Квадратичная сложность. Время выполнения растет пропорционально квадрату размера входных данных.

  6. O(2^n) — Экспоненциальная сложность. Время выполнения удваивается с каждым увеличением размера входных данных.

  7. O(n!) — Факториальная сложность. Время выполнения растет пропорционально факториалу размера входных данных.

Примеры сложности алгоритмов в PHP:

1. O(1) — Константная сложность

php
Copy
function getFirstElement(array $arr) {
    return $arr[0]; // Всегда берет первый элемент, независимо от размера массива
}

Время выполнения не зависит от размера массива.

2. O(n) — Линейная сложность

php
Copy
function findElement(array $arr, $target) {
    foreach ($arr as $element) {
        if ($element === $target) {
            return true;
        }
    }
    return false;
}

В худшем случае алгоритм пройдет по всем элементам массива.

3. O(n²) — Квадратичная сложность

php
Copy
function bubbleSort(array $arr) {
    $n = count($arr);
    for ($i = 0; $i < $n; $i++) {
        for ($j = 0; $j < $n - $i - 1; $j++) {
            if ($arr[$j] > $arr[$j + 1]) {
                // Меняем элементы местами
                $temp = $arr[$j];
                $arr[$j] = $arr[$j + 1];
                $arr[$j + 1] = $temp;
            }
        }
    }
    return $arr;
}

Два вложенных цикла приводят к квадратичной сложности.

4. O(log n) — Логарифмическая сложность

php
Copy
function binarySearch(array $arr, $target) {
    $low = 0;
    $high = count($arr) - 1;

    while ($low <= $high) {
        $mid = (int)(($low + $high) / 2);
        if ($arr[$mid] === $target) {
            return $mid;
        }
        if ($arr[$mid] < $target) {
            $low = $mid + 1;
        } else {
            $high = $mid - 1;
        }
    }
    return -1;
}

На каждом шаге алгоритм уменьшает диапазон поиска вдвое.

5. O(n log n) — Линейно-логарифмическая сложность

php
Copy
function mergeSort(array $arr) {
    if (count($arr) <= 1) {
        return $arr;
    }

    $mid = (int)(count($arr) / 2);
    $left = mergeSort(array_slice($arr, 0, $mid));
    $right = mergeSort(array_slice($arr, $mid));

    return merge($left, $right);
}

function merge(array $left, array $right) {
    $result = [];
    while (count($left) > 0 && count($right) > 0) {
        if ($left[0] < $right[0]) {
            $result[] = array_shift($left);
        } else {
            $result[] = array_shift($right);
        }
    }
    return array_merge($result, $left, $right);
}

Сортировка слиянием делит массив на части и затем объединяет их.

6. O(2^n) — Экспоненциальная сложность

php
Copy
function fibonacci($n) {
    if ($n <= 1) {
        return $n;
    }
    return fibonacci($n - 1) + fibonacci($n - 2);
}

Каждый вызов функции вызывает два дополнительных вызова, что приводит к экспоненциальному росту.

7. O(n!) — Факториальная сложность

php
Copy
function permute($arr, $start = 0) {
    if ($start === count($arr) - 1) {
        echo implode(", ", $arr) . "\n";
    }
    for ($i = $start; $i < count($arr); $i++) {
        // Меняем местами элементы
        list($arr[$start], $arr[$i]) = array($arr[$i], $arr[$start]);
        // Рекурсивно генерируем перестановки
        permute($arr, $start + 1);
        // Возвращаем элементы на место
        list($arr[$start], $arr[$i]) = array($arr[$i], $arr[$start]);
    }
}

Генерация всех перестановок массива имеет факториальную сложность.

Заключение

Понимание сложности алгоритмов помогает выбирать оптимальные решения для задач. Например, для больших данных лучше избегать алгоритмов с квадратичной или экспоненциальной сложностью, отдавая предпочтение линейным или логарифмическим алгоритмам.
