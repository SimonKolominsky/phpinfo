PHP 8.0 представил множество значительных изменений и улучшений. Вот ключевые нововведения:

  1. JIT (Just-In-Time) компиляция - значительно повышает производительность для вычислительно сложных задач
  2. Типизация:
    • Union Types (объединение типов): function foo(int|float $number)
    • Типизированные свойства класса: public string $name;
    • Возвращаемый тип static
    • Ковариантные возвращаемые типы и контравариантные параметры
  3. Именованные аргументы - позволяют передавать аргументы в функции по имени: 
    php
    htmlspecialchars($string, double_encode: false);
  4. Атрибуты (заменяют PHPDoc аннотации): 
    php
    #[Route("/api/posts", methods: ["GET"])]
  5. Конструктор объектов класса без указания new: 
    php
    $datetime = DateTime::createFromFormat(...);
  6. Выражение match (улучшенная замена switch): 
    php
    $result = match($value) {
    1, 2 => 'низкий',
    3, 4 => 'средний',
    default => 'высокий',
};
  7. Оператор Nullsafe: $user?->profile?->name
  8. Строгое сравнение в функции in_array() и array_search() по умолчанию
  9. Новые функции и интерфейсы:
    • Интерфейс Stringable
    • str_contains(), str_starts_with(), str_ends_with()
  10. Обработка исключений:
    • Throw как выражение
    • Новые типы исключений (ValueError)
    • Обнаружение предупреждений как ошибок
  11. Улучшенные сообщения об ошибках с более понятной диагностикой
  12. Устаревание и удаление некоторых возможностей (например, параметра $mandatory функции get_class())

PHP 8.0 фокусируется на безопасности типов, производительности и более чистом синтаксисе, что делает код более надежным и быстрым.

В PHP 8.1 добавлено множество полезных улучшений и новых возможностей. Вот основные изменения:

  1. Перечисления (Enumerations): 
    php
    enum Status {
    case Pending;
    case Active;
    case Suspended;
}
  2. Тип-пересечение (Intersection Types): 
    php
    function foo(A&B $param): void {}
  3. "Никогда" возвращаемый тип (never): 
    php
    function redirect(): never {
    header('Location: /');
    exit();
}
  4. Readonly свойства: 
    php
    class User {
    public readonly string $name;
}
  5. Первоклассная поддержка Callable: 
    php
    $fn = strlen(...);
  6. Новые инициализаторы для массивов: 
    php
    $array = [0, 1, ...$values, 5, 6];
  7. Финальные константы класса: 
    php
    class Foo {
    final const X = "foo";
}
  8. Новый оператор Fibers для асинхронного программирования
  9. Поддержка чистых методов пересечения (pure intersection types)
  10. Новая функция array_is_list() для проверки, является ли массив списком
  11. Новые строковые функции:
    • str_contains()
    • str_starts_with()
    • str_ends_with()
  12. Поддержка использования new в инициализаторах: 
    php
    class Service {
    private Logger $logger = new Logger();
}
  13. Возможность определять octal числа с явным префиксом 0o
  14. Улучшения системы предупреждений и замечаний
  15. Устаревание некоторых функций и возможностей, включая использование неявных вызовов конструкторов в наследуемых классах

PHP 8.1 продолжает линию усовершенствований, начатую в PHP 8.0, с фокусом на улучшение типизации, синтаксического сахара и производительности.

PHP 8.2 вышел в декабре 2022 года и представил несколько важных нововведений и улучшений. Вот основные изменения:

  1. Readonly классы - можно объявить весь класс как readonly: 
    php
    readonly class User {
    public string $name;
    public string $email;
}
  2. Null, false и true как отдельные типы: 
    php
    function foo(): null {}
function bar(): false {}
function baz(): true {}
  3. Снятие ограничений с константных выражений: 
    php
    class Math {
    const PI = 3.14;
    const TWO_PI = self::PI * 2;
}
  4. Тип данных DNF (Disjunctive Normal Form) - комбинация union и intersection типов: 
    php
    function process(A&B|C&D $value) {}
  5. Динамические свойства объявлены устаревшими: 
    php
    $user->undeclaredProperty = 'value'; // Выдаст предупреждение
  6. Улучшенный оператор random_int() и random_bytes() без исключений: 
    php
    $int = mt_rand(min: 1, max: 10);
  7. Улучшения для Closure::callArgs() и array_pad() в сочетании с оператором распаковки
  8. Поддержка mysqli_execute_query() для одношаговых запросов
  9. Улучшения отладки и обработки ошибок
  10. Новые директивы для систем подсказок типов и редакторов: 
    php
    /** @param callable(string, int): bool $callback */
  11. Удаление ряда устаревших функций и возможностей
  12. Улучшение производительности для часто используемых функций

PHP 8.2 сосредоточился на усовершенствовании системы типов, улучшении производительности и завершении некоторых функций, начатых в предыдущих версиях.

PHP 8.3 был выпущен в ноябре 2023 года и включает следующие ключевые улучшения:

  1. Типизированные константы класса: 
    php
    class MathConstants {
    public const float PI = 3.14159;
    protected const string APP_NAME = "MyApp";
}
  2. Новый оператор "typenow" и алиас типа: 
    php
    type UserID = int;
type Point = array{x: int, y: int};

function getUser(UserID $id): User {}
  3. Улучшенный анализ динамических вызовов для повышения безопасности
  4. Новые JSON функции:
    • json_validate() для проверки валидности JSON
    • Улучшенные флаги для json_encode() и json_decode()
  5. Новый класс RandomGenerator для генерации случайных чисел: 
    php
    $random = new \Random\RandomGenerator();
$bytes = $random->getBytes(32);
  6. Инлайн представление перечислений: 
    php
    function handleStatus(Status $status) {
    return match($status) {
        Status::Pending => "Waiting",
        Status::Processing => "In progress",
        /* ... */
    };
}
  7. Оператор "readonly" для аргументов анонимных функций: 
    php
    $fn = function (readonly Point $point) {
    // нельзя изменить $point
};
  8. Улучшения производительности:
    • Оптимизированные функции для строк
    • Улучшенная обработка больших массивов
  9. Улучшенные сообщения об ошибках и предупреждениях
  10. Новые математические функции:
    • cos_tanh()
    • sin_tanh()
    • tan_tanh()
  11. Улучшения для стандартной библиотеки DateTime, включая новые методы для работы с временными зонами
  12. Устаревание некоторых функций:
    • mb_ereg_replace() с модификатором "e"
    • Implicit by-ref in array_multisort()

PHP 8.3 продолжает фокусироваться на улучшении строгой типизации, производительности и безопасности языка.

  14. Расширение функциональности атрибутов (attributes)

PHP 8.4 продолжает улучшать систему типов, производительность и удобство использования языка, следуя общей тенденции PHP к большей строгости типизации и более современному синтаксису.
